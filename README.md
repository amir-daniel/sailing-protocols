# Sailing Protocols — Sailing Checklists & Protocols

A structured, iteratively-built library of sailing checklists and protocols,
inspired by the philosophy in Atul Gawande's *The Checklist Manifesto*.

## Checklist philosophy

Two checklist styles are used throughout this repo:

- **DO–CONFIRM** — Do the tasks from memory/experience, then pause and use the
  list to *confirm* nothing was missed. Best for experienced crew and routine ops.
- **READ–DO** — Read each step and perform it as you go. Best for rare, complex,
  or high-stress procedures (e.g. emergencies).

Good checklists are **short**, focus on the **killer items** (the steps that are
easy to miss and dangerous to skip), and **support** judgment rather than replace it.

Each file marks its intended style near the top.

## Contributing convention

- One checklist per file, kebab-case filename.
- Use the shared template in [TEMPLATE.md](TEMPLATE.md).
- Keep items imperative and scannable. Bold the killer items.
